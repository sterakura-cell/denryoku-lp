# 動力電気代 無料診断LP

法人向け「動力電気代削減 無料診断」のランディングページ（LP）です。
HTML / CSS / JavaScript のみで動作する、スマホファーストの静的サイトです。

最終目的は **条件の良い企業を東京新電力株式会社へ案件連携すること**。
2名体制（寺倉・父）で運用しやすいよう、**電話よりフォーム取得を優先**し、
**S/Aランクの見込み度が高い案件だけ人が対応する**設計にしています。

掲載会社：**東京新電力株式会社**
正規代理店：**エコクリエートホールディングス株式会社**

## 3. URLパラメータで連絡先を切り替える方法

**1つのLPを、URLの `ref` パラメータで2パターンに出し分けます。**
`script.js` がページ読み込み時に `ref` を読み取り、電話番号・紹介元コードを自動でセットします。

| ref値 | 担当 | 表示する電話番号 | 紹介元コード | hidden input |
|-------|------|------------------|--------------|--------------|
| `terakura` | 寺倉 | 090-3698-7711 | terakura | terakura |
| `father`   | 寺倉 | 090-5208-6616 | father   | father   |
| 無し/不正  | 寺倉 | 090-3698-7711 | terakura | terakura（既定） |

連絡先の追加・変更は `script.js` 冒頭の `REF_TABLE` を編集するだけです。

### `index.html?ref=terakura` の使い方（パターンA：寺倉担当用）

寺倉担当が見込み客へ案内するときに使うURL。

```
…/index.html?ref=terakura
```

- 電話番号表示：**090-3698-7711**
- 紹介元コード：**terakura**
- フォーム送信データの `referral_source` に **terakura** が入る

### `index.html?ref=father` の使い方（パターンB：父担当用）

父が既存取引先へ案内するときに使うURL。

```
…/index.html?ref=father
```

- 電話番号表示：**090-5208-6616**
- 紹介元コード：**father**
- フォーム送信データの `referral_source` に **father** が入る

> **どちらのLP経由・どちらの電話番号経由か** は、フォームの hidden input
> `referral_source`（terakura / father）と `landing_url`（実際のアクセスURL）で判別できます。
> 送信データに必ず両方が含まれます。

## 4. フォーム送信機能の接続方法

現状の `script.js` のフォーム送信は **仮実装**です。
送信ボタンを押すと、実送信せずに完了メッセージを表示し、
送信予定データ（ランク・年間電気代・15%/20%削減見込みを含む）を
ブラウザのコンソールに出力します。

本番では以下のいずれかに接続してください。`<form>` の `name` 属性は
そのまま各サービスの項目名として使える命名にしてあります。

### フォームの主な name 属性

| name | 内容 |
|------|------|
| `referral_source` | 紹介元コード（terakura / father）※hidden |
| `landing_url` | 問い合わせ元LPの完全URL ※hidden |
| `company` | 会社名 |
| `contact_name` | 担当者名 |
| `phone` | 電話番号 |
| `email` | メールアドレス |
| `address` | 所在地 |
| `industry` | 業種 |
| `monthly_cost` | 月額の動力電気代 |
| `bill_file` | 電気料金明細アップロード（任意・推奨。後追い送付も可） |
| `after_diagnosis` | 無料診断後の希望（打合せ希望 / 結果だけ） |
| `uses_power` / `cubicle` / `current_provider` / `note` | 任意項目 |

### A. Googleフォームに接続する方法

1. Googleフォームで、上記項目に対応する質問を作成する。
2. フォームの「事前入力したURLを取得」から、各項目の `entry.xxxxxxx` という
   フィールドIDを確認する。
3. `index.html` の各 `name="..."` を、対応する `name="entry.xxxxxxx"` に置き換える。
4. `<form>` を以下に変更する（`/viewform` を `/formResponse` に変える）。

```html
<form id="diagnoseForm" method="POST"
      action="https://docs.google.com/forms/d/e/フォームID/formResponse"
      target="hidden_iframe">
```

5. ファイルアップロードを使う場合、Googleフォームの「ファイルアップロード」設問は
   ログイン必須・直接POST不可のため、**GAS（後述D）経由**を推奨します。

### B. Formspree に接続する方法（実装済み・推奨）

**このLPはFormspree用のAJAX送信を実装済みです。** `script.js` の先頭の定数を設定するだけで本番送信になります。

```js
var FORMSPREE_ENDPOINT = "https://formspree.io/f/あなたのID"; // ここを設定
var FORM_SUPPORTS_FILE = false; // Formspree無料プランは false のまま（ファイル添付不可）
```

- `FORMSPREE_ENDPOINT` が空 → 仮実装（完了画面のみ・未送信）。設定 → 本番送信（画面遷移なし）。
- 送信時に **ランク・年間電気代・15%/20%削減見込み** が隠しフィールドで自動付与され、
  Formspreeのメールにそのまま載ります（管理シート・案件共有に転記しやすい）。
- **Formspree無料プランはファイル添付不可・月50件まで。** `FORM_SUPPORTS_FILE = false` の状態では、
  明細ファイルは送らず「明細あり／なし」のメモだけ送ります（明細は折り返しで受領）。
  有料プランやGASでファイルも受け取る場合のみ `true` にしてください。

詳しい公開手順は [DEPLOY.md](DEPLOY.md) を参照。

### C. HubSpot に接続する方法

1. HubSpotでフォームを作成し、各プロパティを上記 name に合わせる。
2. HubSpot の埋め込みフォーム or Forms API を使用。
   `referral_source` と `landing_url` はカスタムプロパティとして登録すると
   案件管理（CRM）でそのまま絞り込めます。

### D. Google Apps Script（GAS）に接続する方法（明細アップロード対応・推奨）

明細ファイルを Google ドライブに保存し、データをスプレッドシートに自動追記できます。

1. スプレッドシートを作成 → 拡張機能 → Apps Script。
2. `doPost(e)` で受信データをシートに追記し、ファイルを Drive に保存する関数を書く。
3. 「デプロイ → ウェブアプリ」で公開し、`/exec` URLを取得。
4. `script.js` の送信処理を `fetch(GAS_URL, { method:"POST", body:new FormData(form) })`
   に置き換え、成功時に完了画面を表示する。

> いずれの方式でも、`referral_source` と `landing_url` を必ず保存先に含めてください。
> これが「どちらの紹介元・どちらのLP経由か」の判別キーになります。

## 5. Googleスプレッドシート 管理項目案

受信した案件は、以下の項目でスプレッドシート管理することを推奨します。

| 列 | 項目 | 備考 |
|----|------|------|
| A | 受付日 | 自動 or 手入力 |
| B | 紹介元コード | terakura / father |
| C | 問い合わせ元LP URL | `landing_url` |
| D | 会社名 | |
| E | 担当者名 | |
| F | 電話番号 | |
| G | メールアドレス | |
| H | 所在地 | |
| I | 業種 | |
| J | 月額動力電気代 | |
| K | 年間電気代 | = J×12 |
| L | 15％削減見込み | = K×0.15 |
| M | 20％削減見込み | = K×0.20 |
| N | 明細提出 | あり / なし |
| O | 打合せ希望 | あり / なし |
| P | 見込み度ランク | S / A / B / C / 対象外 |
| Q | 対応状況 | 未対応 / 明細依頼中 / 診断中 / 電話済 / 東京新電力へ連携済 / 成約 / 失注 |
| R | 次回対応日 | |
| S | メモ | |

> K〜M列はスプレッドシートの数式で自動計算できます。
> ランク（P列）は `=IF(J>=500000,"S",IF(J>=200000,"A",IF(J>=100000,"B",IF(J>0,"C","対象外"))))`。

## 6. 紹介元コードの管理方法

- **terakura** … 寺倉担当（TEL 090-3698-7711）経由の問い合わせ
- **father** … 父担当（TEL 090-5208-6616）経由の問い合わせ。既存取引先向け。

運用ルール：

1. 案内する相手によって **配布するURLを使い分ける**（`?ref=terakura` / `?ref=father`）。
2. スプレッドシートの「紹介元コード」列で集計すれば、どちらの導線が成果につながったか分かる。
3. コードを増やしたい場合（例：別ルート）、`script.js` の `REF_TABLE` に1行追加するだけ。

## 7. 見込み度ランクの運用方法

月額の動力電気代でランクを自動判定します。

| ランク | 条件（月額動力電気代） | 対応方針 |
|--------|------------------------|----------|
| **S** | 50万円以上 | **最優先**。明細到着後すぐ電話 → 東京新電力へ連携 |
| **A** | 20万円以上 | 優先。明細確認後に電話対応 |
| **B** | 10万円以上 | 明細をもとに診断、メール中心で対応 |
| **C** | 10万円未満 | 明細確認のうえ、余力があれば対応 |

**2名体制を回すコツ**：
- 人が電話するのは原則 **S / A ランクのみ**。B / C はフォーム・メールで完結を目指す。
- LP上でも入力中にランクが表示され、Sランクには「優先的にご連絡します」と案内済み。

## 8. 2名体制での運用手順

1. **配布**：寺倉は `?ref=terakura`、父は既存取引先へ `?ref=father` のURLを案内。
2. **受信**：フォーム送信データ（紹介元・LP URL・ランク・削減見込みを含む）をスプレッドシートに集約。
3. **仕分け**：ランク列で並べ替え、**S → A** の順に対応。
   明細アップロードは任意のため、**未提出の案件は「明細依頼中」**にして、一次連絡時に必ず明細を依頼する（明細がないと正式診断・東京新電力連携に進めないため）。
4. **一次対応**：S/Aは担当（寺倉/父）が電話で確認。B/Cはメールで診断結果を案内。
5. **連携**：条件が良くて打合せ希望 or 明細あり の案件を **東京新電力へ案件共有**（次項フォーマット）。
6. **更新**：対応状況・次回対応日を更新し、取りこぼしを防ぐ。

担当の振り分け目安：`referral_source = terakura` → 寺倉、`= father` → 父が一次対応。

## 10. 表現に関する注意（禁止表現）

LP・案内・連携時に、以下の断定表現は**使用しません**。

- 「必ず安くなる」「必ず20％削減」「保証」「最大削減保証」
- 「誰でも対象」「絶対に下がる」「契約しないと損」「今すぐ契約してください」

代わりに **「削減できる可能性があります」** と表現します。
LP本文・FAQ・シミュレーター注記も、この方針で統一済みです。
